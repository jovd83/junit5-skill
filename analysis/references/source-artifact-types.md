# Source Artifact Types

1. Requirements usually describe expected behavior and acceptance criteria.
2. Code usually reveals current behavior, side effects, guard rails, and exception paths.
3. Defect reports usually reveal regression targets and missing negative coverage.
4. TDD cases usually map well to concise input, action, and expected-result tests.
5. BDD scenarios usually map well to named scenarios, nested groups, or parameterized tables.
6. Plain-text definitions often need normalization before they become executable tests.
