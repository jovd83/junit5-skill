# Source Mapping

1. Requirement text maps to externally observable behavior and acceptance criteria.
2. Code analysis maps to public methods, side effects, boundary conditions, and failure paths.
3. TDD narratives map well to focused unit or slice tests with explicit expected outcomes.
4. BDD scenarios map well to named test methods, nested structures, or parameterized cases.
5. Plain-text test definitions should be normalized into setup, action, and expected result.
6. Keep a clear distinction between source-backed scenarios and inferred risk-based additions.
