# Capability Map

- `core/`: Create, run, debug, evaluate, refactor, and correct JUnit 5 tests.
- `analysis/`: Derive executable scenarios from requirements, code, or narrative test definitions.
- `architecture/`: Shape package layout, ownership, fixtures, and naming.
- `ci/`: Control Maven, Gradle, Console Launcher, tags, and CI behavior.
- `migration/`: Move legacy suites from JUnit 3 or JUnit 4 to JUnit 5.
- `extensions/`: Build reusable callbacks, resolvers, and custom annotations.
- `documentation/tests/`: Explain suite and scenario intent.
- `documentation/root_cause/`: Diagnose failing runs.
- `reporting/stakeholder/`: Summarize outcomes for non-technical audiences.
- `installers/`: Help IDE setup for local development.
